OANP — Open Agent Negotiation Protocol

v0.1 draft · publishing 2026-04-22

An open protocol for fair negotiation between AI agents in commerce.

The gap

MCP, ACP, and UCP solve discovery and payment for agentic commerce. When a buyer agent negotiates with a merchant agent, there is no protocol-level guarantee that the stronger model doesn't systematically exploit the weaker one.

OANP specifies a three-party flow — buyer, arbiter, merchant — with seven mechanically checkable fairness invariants and a signed final agreement. It's designed to extend existing stacks, not replace them.

Designed to coexist

  • MCP Anthropic — discovery
  • ACP OpenAI + Stripe — payment
  • UCP Google — commerce envelope
  • A2A Google — transport
  • OANP — fairness layer between discovery and payment

Status

v0.1 draft is under review at oanp-protocol/spec#1. Final publication is targeted for April 22, 2026. A reference implementation maintained by Valyzen is mapped at oanp-protocol/reference.
